Expand description

A Session containing geometry objects with hierarchical and graph structures.

The Session serves as a container for managing geometry objects (currently Points) along with their relationships through tree and graph data structures. It provides JSON serialization capabilities for cross-language interoperability.

Fields§

§guid: String

Unique identifier for the session

§name: String

Human-readable name for the session

§objects: Objects

Collection of geometry objects (Points)

§lookup: HashMap<String, Geometry>

Lookup table mapping object GUIDs to geometry objects (fast heterogeneous lookup)

§tree: Tree

Hierarchical tree structure for organizing objects

§graph: Graph

Graph structure for representing object relationships

§bvh: BVH

Boundary Volume Hierarchy for spatial collision detection

§cached_ray_bvh: Option<BVH>

Cached BVH for ray casting (indices map to cached_guids)

§cached_guids: Vec<String>

Cached GUIDs corresponding to cached_boxes order

§cached_boxes: Vec<BoundingBox>

Cached AABBs for ray-casting BVH

§bvh_cache_dirty: bool

Dirty flag for cached ray BVH

pub fn get_geometry(&self) -> Objects

Get all geometry with transformations applied from tree hierarchy.

Recursively traverses the tree and applies parent transformations to children. Each child’s transformation is the composition of all ancestor transformations multiplied by its own transformation.

§Returns
